CLI (Command Line Interface) - це інтерфейс командного рядка, текстовий спосіб взаємодії з комп'ютером або програмним забезпеченням, де команди вводяться у вигляді текстових рядків. Замість графічного інтерфейсу, користувач взаємодіє з системою шляхом введення команд, які обробляються спеціальним інтерпретатором, що називається оболонкою.
- `pytest -q` — тихий режим (крапками ...)
- `pytest -qq` — ще меньше інформації
- `pytest -k "test_qa"` — слово по якому можемо шукати тести
- `pytest --ff` — першим виводить непрацюючий тест
- `pytest --maxfail=2` — вказуєш через яку кількисть непрацючих тестів зупинити запуск (2)
- `pytest --durations=2` — показує (2) тест який працює найповільніше (тормозить) для цього треба `import time` → `time.sleep(1)`
- `pytest -vs` (-s) — виводити print в тесті
- `pytest --tb=no` — вимикає traceback (помилки)
- `pytest --tb=no -q` — вимикає traceback в тихому режимі
- pytest --collect-only -  не запускає тести, а лише **збирає їх** і показує структуру: які файли, які функції, які класи він знайшов.

# Pytest CLI Cheatsheet

| Option / Command            | Description                                                                                                                                                                                                                                                                          |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `-v`                        | Verbose mode — докладний вивід (які тести запускаються)                                                                                                                                                                                                                              |
| `-q`                        | Quiet mode — коротший вивід                                                                                                                                                                                                                                                          |
| `-qq`                       | Extra quiet — тільки помилки                                                                                                                                                                                                                                                         |
| `-k "keyword"`              | Запуск тільки тих тестів, де в назві є `keyword`                                                                                                                                                                                                                                     |
| `-m "marker"`               | Запуск тестів з певним маркером (`@pytest.mark.smoke`)                                                                                                                                                                                                                               |
| `-m "smoke and regression"` | Запуск тільки тестів з обома маркерами                                                                                                                                                                                                                                               |
| `-m "smoke or regression"`  | Запуск тестів з будь-яким з маркерів                                                                                                                                                                                                                                                 |
| `-m "not regression"`       | Запуск усіх, крім `regression`                                                                                                                                                                                                                                                       |
| `--ff`                      | First fail — першим запускає останній непрацюючий тест                                                                                                                                                                                                                               |
| `--maxfail=2`               | Зупинити після 2-х невдалих тестів                                                                                                                                                                                                                                                   |
| `--durations=2`             | Показати 2 найповільніших тести                                                                                                                                                                                                                                                      |
| `-s` або `-vs`              | Виводити `print` з тестів у консоль                                                                                                                                                                                                                                                  |
| `--tb=no`                   | Вимкнути traceback у виводі помилок                                                                                                                                                                                                                                                  |
| `--tb=no -q`                | Traceback off + quiet mode                                                                                                                                                                                                                                                           |
| `--tb=auto`                 | Автоматичний режим traceback (за замовчуванням)                                                                                                                                                                                                                                      |
| `--tb=short`                | Скорочений traceback                                                                                                                                                                                                                                                                 |
| `--tb=line`                 | Однорядковий traceback                                                                                                                                                                                                                                                               |
| `--tb=long`                 | Повний traceback                                                                                                                                                                                                                                                                     |
| `--junitxml=report.xml`     | Зберегти результати у форматі XML                                                                                                                                                                                                                                                    |
| `--setup-show`              | Показати, як працюють фікстури (setup/teardown)                                                                                                                                                                                                                                      |
| `-n 5`                      | Запустити тести у 5 потоків (паралельно, потрібен pytest-xdist)                                                                                                                                                                                                                      |
| `-n auto`                   | Автоматично вибрати кількість потоків (pytest-xdist)                                                                                                                                                                                                                                 |
| `--capture=method`          | Вибір способу перехоплення виводу: `fd` \| `sys` \| `no`                                                                                                                                                                                                                             |
| `-l` або `--showlocals`     | Показати локальні змінні у traceback                                                                                                                                                                                                                                                 |
| `--collect-only`            | Тільки зібрати тести, не виконуючи їх                                                                                                                                                                                                                                                |
| `--version`                 | Показати версію pytest та інформацію про імпорт                                                                                                                                                                                                                                      |
| `-h` або `--help`           | Показати довідку та конфігураційні опції                                                                                                                                                                                                                                             |
| -`x`                        | зупиняє виконання одразу після першого фейлу (дуже зручно, якщо шукаєш одну конкретну проблему).                                                                                                                                                                                     |
| `-r`chars                   | show extra test summary info as specified by chars<br>(f)ailed, (E)error, (s)skipped, (x)failed, (X)passed,<br>(p)passed, (P)passed with output, (a)all except pP.<br>It’s not only helpful for understanding test skips, but also you can use it for other test outcomes<br>as well |

## Running specific tests
| Command | Description |
|---------|-------------|
| `pytest test_file.py` | Запустити тести з конкретного файлу |
| `pytest test_file.py::TestClass` | Запустити всі тести з класу |
| `pytest test_file.py::TestClass::test_name` | Запустити конкретний тест |
